This repo is made while implementing different kinds of RAGs, strating from simple (Naive) RAG.  

These RAGs will be implemented,

1. Naive RAG (single PDF as a source) + Evals
2. Naive RAG with multiple docs
3. Naive RAG (Python and C code as a source, multiple files)
4. Naive RAG with tabular data, OCR, multiple files
5. Experiment with [document retrievers](https://docs.langchain.com/oss/python/integrations/retrievers)
6. Explore Chonkie for semantic based chunking
7. RAG for whole repo or knowledge base level
8. GraphRAG / Hierarchial RAG
9. Hybrid RAG

Alongside, we'll learn 

- Different chunking strategies
- RAG evaluation
- Reranking
- Query rewriting

many more
